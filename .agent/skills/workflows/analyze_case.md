# OB Case Analysis Workflow

Use this workflow when the user provides an organizational scenario or case for analysis.

## Steps

1. **Read the case carefully** and identify:
   - The organization and industry
   - The key people / roles involved
   - The timeline and context

2. **Identify the core problem(s)** — what OB issues are present? Common categories:
   - Motivation issues
   - Leadership failures
   - Team dysfunction
   - Communication breakdown
   - Cultural misalignment
   - Change resistance
   - Power / political conflict

3. **Select 2-3 relevant OB theories** from `SKILL.md`'s Theory Mapping tables that best explain the situation.

4. **Write the analysis** using the Case Analysis Template from `rules.md`:

   ```
   ## Case: [Case Title]
   1. **Background**: Brief context
   2. **Problem Identification**: Core issue(s)
   3. **Theory Application**: Which OB theories explain the situation
   4. **Analysis**: Detailed application of theory to facts
   5. **Recommendation**: Practical suggestions
   6. **Key Takeaway**: One-sentence lesson
   ```

5. **Provide bilingual key terms** — list all OB terms used in the analysis with Chinese translations.

6. **Save the output** to `cases/Case_[Title].md` (create `cases/` folder if it doesn't exist).

## Tips

- Always apply at least **two different theories** to show depth of analysis
- Recommendations should be **specific and actionable**, not generic
- Use **academic English** suitable for undergraduate coursework
- Include a brief **model answer** section showing how to write this as an exam response
